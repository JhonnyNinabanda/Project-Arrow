\subsection{Requisitos}

\subsubsection*{Requerimientos de desarrollo}

El equipo de desarrollo requiere Unity 6 LTS, un editor de código compatible con C\#, Git, Git LFS y acceso al repositorio de GitHub. Se recomienda utilizar una versión común del motor para evitar incompatibilidades en escenas, prefabs y paquetes.

\subsubsection*{Requerimientos funcionales}

El prototipo debe incluir movimiento lateral, salto, combate cuerpo a cuerpo, ataque a distancia, enemigos con IA básica, sistema de salud, recolección de Diamantes de Almas, HUD, checkpoints, menús principales y al menos una condición de victoria o derrota.

\subsubsection*{Requerimientos no funcionales}

El juego debe mantener una tasa de cuadros estable en equipos de gama media, ofrecer controles responsivos, presentar una interfaz legible en resoluciones comunes y conservar una organización de proyecto comprensible para el equipo.

\subsubsection*{Configuración de repositorio}

El archivo \texttt{.gitignore} debe estar configurado para Unity, ignorando carpetas como \texttt{Library}, \texttt{Temp}, \texttt{Obj}, \texttt{Builds} y otros archivos generados localmente. Los activos binarios pesados deben rastrearse mediante Git LFS.

\subsubsection*{Criterios mínimos de entrega}

La entrega académica debe incluir ejecutable funcional para Windows, repositorio organizado, documentación del GDD, evidencia de control de versiones, capturas o video de demostración y descripción de riesgos y mitigaciones.
